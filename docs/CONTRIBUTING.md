## How to contribute to [[REPO_NAME]]

#### **Did you find a bug?**

* Search to see if the bug was already reported under [Issues](https://github.com/[[USERNAME]]/[[REPO_NAME]]/issues).
* If not, [open a new issue](https://github.com/[[USERNAME]]/[[REPO_NAME]]/issues/new) with a clear title and description.

#### **Did you write a patch that fixes a bug?**

* Fork the repository, create a branch, and open a pull request with your patch.
* Use a title prefix such as `fix:` for bug fixes.

#### **Do you intend to add a new feature or change an existing one?**

* Fork the repository, create a branch, and open a pull request.
* Prefix the PR title with the appropriate type
* Labels will be applied automatically based on the title.

| Title       | Applied Label   | Description                                  |
|-------------| --------------- | -------------------------------------------- |
| `feat:`     | `feature`       | Adds a new feature or capability             |
| `fix:`      | `bug`           | Fixes a bug or incorrect behavior            |
| `docs:`     | `documentation` | Documentation-only changes                   |
| `style:`    | `enhancement`   | Code style changes (no logic changes)        |
| `refactor:` | `refactor`      | Code restructuring without functional change |
| `perf:`     | `enhancement`   | Performance improvements                     |
| `test:`     | `enhancement`   | Adding or modifying tests                    |
| `build:`    | `enhancement`   | Build system or dependency changes           |
| `ci:`       | `ci`            | CI/CD configuration changes                  |
| `chore:`    | `chore`         | Maintenance tasks or housekeeping            |

#### **Do you have questions about the source code?**

* Ask any question about how to use [[REPO_NAME]] by [Email](mailto:[[EMAIL]]).
